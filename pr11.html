Finding optimal weights for Neural Networks¶

Back propagation versus Ramdomized optimization¶

1. Objective:¶

The purpose of this project is to find the optimal weights for Neural Network using Randomized Optimization and compare the perfromance against the perfromance of Neural Network with back propagation.

2. Datasets¶

Wine Dataset¶

UCI Wine Quality dataset consists physicochemical and sensory data red and white variants of the Portuguese "Vinho Verde" wine. Dataset is available at https://archive.ics.uci.edu/ml/datasets/Wine+Quality

Number of Instances: red wine - 1599
Number of Attributes: 11 + output attribute

Attribute Information Input variables (based on physicochemical tests): 1 - fixed acidity 2 - volatile acidity 3 - citric acid 4 - residual sugar 5 - chlorides 6 - free sulfur dioxide 7 - total sulfur dioxide 8 - density 9 - pH 10 - sulphates 11 - alcohol
Output variable (based on sensory data): 12 - quality (score between 0 and 10)

model_stats = []
for mi in [100, 500, 1000]:
    for ma in [10, 50, 100]:
        for lr in [0.001, 0.01, 1]:
            
            nn_rhc_clf = mlrose.NeuralNetwork(hidden_nodes = [30], activation = 'relu',
                                         algorithm = 'random_hill_climb', max_iters = mi,
                                         bias = True, is_classifier = True, learning_rate = lr, 
                                         max_attempts= ma, random_state = 11)

            t_start = time.time()*1000.0
            nn_rhc_clf = nn_rhc_clf.fit(Wine_X_train, Wine_y_train)
            t_stop = time.time()*1000.0

            t_start = time.time()*1000.0
            Wine_y_pred = cross_val_predict(nn_rhc_clf, Wine_X_train, Wine_y_train, cv=cv, n_jobs=-1)

            prscore = precision_score(Wine_y_train, Wine_y_pred, average='binary')
            rcscore = recall_score(Wine_y_train, Wine_y_pred, average='binary')
            f1score = f1_score(Wine_y_train, Wine_y_pred, average='binary')

            # MI=100, MA=10, LR=0.001
            model_stats.append(['RHC', mi, ma, lr, "{:.2f}".format(prscore), "{:.2f}".format(rcscore), "{:.2f}".format(f1score), traintime, testtime])

8 Conclusion¶

As expected, Back propagation was much faster as compared to all optimization algorithms because back propagation adjusts the weights in the right direction based on actual output whereas optimization algorithms relies of random changes and do not leverage available information.
Backpropagation tries to minimize error by learning the error and correcting it in each iteration. Neural network with back propagation is adaptive to the error and hence faster than GA.
Performance of Genetic algorithm was better compared to all other methods. It takes advantage of random selection, mutation and cross over to avoid being caught in a local optimum. Performance of back propagation is lower mostly likely because it can get stuck in local optima.
