Data Format: For each stock typically stock data is available in CSV format with columns -

Date, Open, High, Low, Close, Volume, Adj Close

Note that Close column is the actual price that was reported at the exchange when the stock closed for that day and Adjusted Close is the adjusted price due to events such as stock splits and dividend payments etc. We use pandas dataframe in order to process this stock data in python.

Pandas functions df.dropna() is used to remove rows with NaN values and df.dropna(subset=COLNAME) is used to remove rows with NaN values in specified column

We also use df.join to combine data from two stocks into a single dataframe. df.join(otherdf, how=inner) executes the inner join, only includes data where both exist in df and other df. It also eliminates all the bad dates when stock market is closed.

Statistical Analysis of Time Series:

Rolling Statistics: A Bollinger Band is a technical analysis tool defined by a set of lines plotted two standard deviations (positively and negatively) away from a simple moving average (SMA) of the security's price, but can be adjusted to user preferences. Bollinger Bands were developed and copyrighted by famous technical trader John Bollinger.

Daily Return: Daily return is the profit/loss made by the stock compared to the previous day.

Cumulative Return: While daily returns are useful, it doesn't give immediate insight into the gains he had made till date, especially if the stock is very volatile. Cumulative return is computed relative to the day investment is made. If cumulative return is above one, you are making profits else you are in loss.

Histogram of Daily Returns: Kurtosis tells us about the tails of a distribution. If we assume that our histogram is similar to a gaussian Distribution or normal distribution, then the measure of Kurtosis tells us how different our histogram is from that of the traditional distribution.
If we measure a positive Kurtosis (fat tails) it means that there are more occurrences out in these tails than there are in the normal distribution.
If we measure a negative Kurtosis (skinny tails) it means that there are fewer occurrences out in these tails than there are in the normal distribution

Volatility is measured by the standard deviation of daily returns displayed using Histogram. Volatile stocks have return distributions that are very spread out. By contrast, low volatility stocks have tight distributions, where the daily returns are clustered around the average.

Scatter Plots: (the daily returns of that stock with respect to SPY) Beta - how reactive is the stock to the market. If the slope (Beta) is 1 it means that if the market goes up 1%, that stock also goes up 1%. If beta is 2, if the market goes up 1%, the stock goes up 2%. Alpha - That stock on average is performing a little bit better than the S&P500 (SPY) every day if Alpha is positive. If its negative it means its returned a little less than the market overall.

The slope (Beta) is not Correlation! Correlation is how tightly these individual points fit that line. So you could have a shallow slope with a higher correlation, or you can have a steeper slope and the data fitting the line at a higher correlation.

Portfolio Statistics: To assess the performance of portfolio following statistics are tracked -

  • Cumulative return
  • Average period return (if sampling frequency == 252 then this is average daily return)
  • Standard deviation of daily returns
  • Sharpe ratio of the overall portfolio, given daily risk free rate (usually 0), and yearly sampling frequency (usually 252, the no. of trading days in a year)
  • Ending value of the portfolio

Portfolio Optimization: The simplest way to optimize the portfolio is to compare Cumulative return of stocks that maximized return over that time period. The allocation would be 100% to the highest returning stock.

But if you want to optimize for minimum volatility or Sharpe ratio you have to evaluate various combinations of those stocks. In order to use an optimizer that minimizes we need to do the following:

  • Provide a function to minimize f(x)
  • Provide an initial guess for x
  • Call the optimizer
  • And we want the optimizer to try different allocations to discover the best set of allocations that optimizes this function
  • We want to optimize for Sharpe ratio, but f(x) is not the actual Sharpe ratio value, we want to optimize for -1 * the largest Sharpe ratio because the available optimizer function is minimize
  • This will find the best allocation or the best value for x to solve this problem
  • x can have multiple dimensions and each of the dimensions is the percentage of funds to allocate to each of those stocks

#Function to optimize
def get_negative_sharpe_ratio(allocs, prices, sv=1000000, rfr=0.0, sf=252):
    # Get daily portfolio value
    # Normalize the prices according to the first day
    norm_prices = prices/prices.iloc[0,:]

    # Compute prices based on the allocations
    alloc_prices = norm_prices * allocs

    # Calculate position values
    pos_vals = alloc_prices * sv

    # Get daily portfolio value
    port_val = pos_vals.sum(axis=1).to_frame()
    port_val.columns = ["port_val"]

    # Get portfolio statistics
    cr = port_val.iloc[-1, 0]/port_val.iloc[0, 0] - 1

    #Compute daily returns
    daily_returns = port_val.pct_change()
    daily_returns.iloc[0,:] = 0
    daily_returns = daily_returns[1:]

    adr = daily_returns["port_val"].mean()
    sddr = daily_returns["port_val"].std()
    sr = np.sqrt(sample_freq) * (adr - risk_free_rate) / sddr
    neg_sr = sr * (-1)

    return neg_sr

bounds = ((0,1),(0,1),(0,1),(0,1))

constraints = ({'type': 'eq', 'fun': lambda x:  np.sum(x)-1.0})

initial_guess = np.ones((len(['GOOG', 'IBM', 'GLD', 'AAPL'])), dtype=np.float32)/(len(['GOOG', 'IBM', 'GLD', 'AAPL']))

result = spo.minimize(get_negative_sharpe_ratio, initial_guess, args=(prices,), method='SLSQP', constraints=constraints, bounds=bounds)

Choosing a portfolio from among the enormous range of assets now available to an investor would be facilitated if we could locate the return risk ratio of a particular allocation along a spectrum of possibilities. A comparison between portfolio choices can tell us, for example, whether it is better to select a suboptimal portfolio from a large class of assets or to perform a Markowitz optimal procedure on a subset of the assets. A common criterion for this assessment is the expected return-to-risk tradeoff as measured by the Sharpe ratio.
