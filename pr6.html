rmse_insample=[]
rmse_outsample=[]
for i in range (50):
    dtlearner = dt.DTLearner(leaf_size = i, verbose =False)
    dtlearner.addEvidence(trainX, trainY)
    predY = dtlearner.query(trainX)
    rmse_insample.append(math.sqrt(((trainY - predY) ** 2).sum() / trainY.shape[0]))
    predY = dtlearner.query(testX)
    rmse_outsample.append(math.sqrt(((testY - predY) ** 2).sum() / testY.shape[0]))
    
plt.xlabel("Leaf Size")
plt.ylabel("RMSE")
plt.plot(rmse_insample,label="InSample")
plt.plot(rmse_outsample, label="OutSample")
plt.title("Overfitting Vs Leaf Size for DTLearner")
plt.xticks(range(0,50,5))
plt.grid(True)
plt.legend()
plt.show()

Analysis: This experiment was performed to observe how leaf size affect the overfitting in Decision Tree Leaners. This experiment uses RMSE to evaluate overfitting of the model. Overfitting occurs when model fits training data. The experiment was carried out on Istanbul dataset and bagging was not implemented. Dataset was divided into Training set (60%) and Testing set (40%).
In this experiment RMSE for DT Leaner was calculated for different Leaf Sizes (1 to 50). Since the dataset contains timeseries data, dataset is shuffled to avoid skewing. Random seed of 123 was used shuffling the data. Experiment was repeated 20 times at each leaf to smooth out the observations.
Below plot depicts as experiment starts with smaller leaf sizes difference in the RMSE for train and test data is higher and it reduces with increase in the leaf sizes. In this experiment overfitting occurs at leaf sizes less than 20.

Conclusion: It can be concluded that overfitting decreases with increase in leaf size.

rmse_insample=[]

for i in range (50):
    itr_rmse_insample=[]
    itr_rmse_outsample=[]
    for j in range(20):
        bglearner = bl.BagLearner(learner = dt.DTLearner, kwargs = {"leaf_size":i}, bags = 20, boost = False, verbose = False)
        bglearner.addEvidence(trainX, trainY)
        predY = bglearner.query(trainX)
        itr_rmse_insample.append(math.sqrt(((trainY - predY) ** 2).sum() / trainY.shape[0]))
        predY = bglearner.query(testX)
        itr_rmse_outsample.append(math.sqrt(((testY - predY) ** 2).sum() / testY.shape[0]))
    rmse_insample.append(sum(itr_rmse_insample)/20)
    predY = bglearner.query(testX)
    rmse_outsample.append(sum(itr_rmse_outsample)/20)
plt.xlabel("Leaf Size")
plt.ylabel("RMSE")
plt.plot(rmse_insample,label="InSample")
plt.plot(rmse_outsample, label="OutSample")
plt.title("Overfitting Vs Bagging & Leaf Size")
plt.xticks(range(0,50,5))
plt.grid(True)
plt.legend()
plt.show()

Analysis: This experiment was performed to observe how bagging affect the overfitting in Decision Tree Learners. This experiment uses RMSE to evaluate overfitting of the model. Overfitting occurs when model fits training data. The experiment was carried out on Istanbul dataset and bagging was implemented with 20 bags for each iteration. Dataset was divided into Training set (60%) and Testing set (40%).
In this experiment RMSE for DT Leaner was calculated for different Leaf Sizes (1 to 50). Since the dataset contains timeseries data, dataset is shuffled to avoid skewing. Random seed of 123 was used shuffling the data. Experiment was repeated 20 times at each leaf to smooth out the observations.
From below plots it can be observed that bagging does not eliminates the overfitting but it does reduce the overfitting. When compared to previous experiment, it can be noted that with bagging difference between train and test RMSE is not as large and RMSE for InSample data starts between 0.002 and 0.003.

Conclusion: It can be concluded that Bagging reduces overfitting but does not eliminate it

dtTrainTime = []
dtQueryTime = []
dtNodeCnt = []
rtTrainTime = []
rtQueryTime = []
rtNodeCnt = []


for i in range (50):
    dtItrTrainTime = []
    dtItrQueryTime = []
    dtItrNodeCnt = []
    rtItrTrainTime = []
    rtItrQueryTime = []
    rtItrNodeCnt = []
    for j in range(20):
        startTime = time.time()
        dtlearner = dt.DTLearner(leaf_size = i, verbose =False)
        dtlearner.addEvidence(trainX, trainY)
        dtItrTrainTime.append(time.time()-startTime)

        startTime = time.time()
        predY = dtlearner.query(trainX)
        dtItrQueryTime.append(time.time()-startTime)
        dtItrNodeCnt.append(dtlearner.tree.shape[0])

        startTime = time.time()
        rtlearner = rt.RTLearner(leaf_size = i, verbose =False)
        rtlearner.addEvidence(trainX, trainY)
        rtItrTrainTime.append(time.time()-startTime)

        startTime = time.time()
        predY = rtlearner.query(trainX)
        rtItrQueryTime.append(time.time()-startTime)
        rtItrNodeCnt.append(rtlearner.tree.shape[0])

    dtTrainTime.append(sum(dtItrTrainTime)/20)
    dtQueryTime.append(sum(dtItrQueryTime)/20)
    dtNodeCnt.append(sum(dtItrNodeCnt)/20)
    rtTrainTime.append(sum(rtItrTrainTime)/20)
    rtQueryTime.append(sum(rtItrQueryTime)/20)
    rtNodeCnt.append(sum(rtItrNodeCnt)/20)

plt.xlabel("Leaf Size")
plt.ylabel("Time")
plt.plot(dtTrainTime,label="DT Learning Time")
plt.plot(rtTrainTime, label="RT Learning Time")
plt.plot(dtQueryTime,label="DT Query Time")
plt.plot(rtQueryTime, label="RT Query Time")
plt.title("DT Vs RT - Learning & Query Time")
plt.xticks(range(0,50,5))
plt.grid(True)
plt.legend()
plt.show()

Analysis: This experiment was performed to quantitatively compare "classic" decision trees (DTLearner) versus random trees (RTLearner) to determine which method is better than other. Experiment was performed to measure and compare “Learning and Query Time” and “Tree Size” The experiment was carried out on Istanbul dataset and bagging was not implemented. Dataset was divided into Training set (60%) and Testing set (40%).
In this experiment Learning & Query Time and Tree Size for DT & RT Leaner was calculated for different Leaf Sizes (1 to 50). Since the dataset contains timeseries data, dataset is shuffled to avoid skewing. Random seed of 123 was used shuffling the data. Experiment was repeated 20 times at each leaf to smooth out the observations.
Learning and Query Time: For smaller leaf size, learning time for DT Learner is significantly higher as compared to RT Learner. Learning time for DT Learner reduces as leaf size increases and reaches close to RT Learner for leaf size higher than 40. It was also observed that query time for DT Learner and RT Leaner is not significantly different
